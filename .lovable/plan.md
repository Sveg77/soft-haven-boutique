

## План: 18 товаров (по 3 в каждой категории) с фото и характеристиками

### Текущее состояние
- 6 категорий — ок
- 8 товаров, распределены неравномерно. Нужно 18.

### Шаг 1. Обновить характеристики у существующих 8 товаров

Каждый товар получит 3 цвета, 3 размера и 3 материала в `characteristics`:

```json
{
  "Цвет": ["Лавандовый", "Кремовый", "Серый"],
  "Размер": ["1,5-спальный", "2-спальный", "Евро"],
  "Материал": ["Сатин", "Поплин", "Бязь"]
}
```

### Шаг 2. Добавить 10 новых товаров (до 3 в каждой категории)

| Категория | Есть | Нужно добавить |
|-----------|------|----------------|
| Постельное бельё | 2 | 1 |
| Подушки и одеяла | 2 | 1 |
| Шторы | 1 | 2 |
| Покрывала | 1 | 2 |
| Полотенца | 1 | 2 |
| Аксессуары | 1 | 2 |

Каждый новый товар — с ценой, описанием, и характеристиками (3 цвета, 3 размера, 3 материала).

### Шаг 3. Сгенерировать фото для всех 18 товаров

- Использовать AI (google/gemini-2.5-flash-image) для генерации изображений
- Загрузить в storage bucket `product-images`
- Обновить `image_url` для каждого товара

### Шаг 4. Сгенерировать фото для 6 категорий

- Также через AI, загрузить в storage и обновить `image_url` в `categories`

### Технические детали
- Данные обновляются через insert tool (UPDATE/INSERT SQL)
- Изображения генерируются скриптом через AI Gateway и загружаются в Supabase Storage через curl
- Характеристики хранятся как JSON в поле `characteristics`
