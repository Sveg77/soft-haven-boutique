

## План: Расширение админ-панели

### 1. Новые статусы заказов

Добавить `shipped` (Передан на доставку) и `delivering` (Доставка) в `statusLabels`/`statusColors` и во все Select-компоненты статусов в `Orders.tsx`.

### 2. Поиск по имени/телефону

Добавить поле `Input` с поиском в шапку страницы заказов. Фильтрация на клиенте по `customer_name` и `phone` (includes, case-insensitive).

### 3. Удаление заказа

- Миграция: добавить RLS-политику DELETE на таблицу `orders` для authenticated (аналогично UPDATE).
- Добавить кнопку удаления в строку таблицы и в диалог деталей с подтверждением через `AlertDialog`.
- Мутация: удаляет `order_items` по `order_id`, затем `orders`.

### 4. Архив

Добавить статус `archived`. Кнопка «В архив» в деталях заказа переводит статус в `archived`. В фильтре добавить «Архив». По умолчанию архивные заказы скрыты (фильтр «Все» не включает `archived`).

### 5. Дашборд — новая страница `/admin/dashboard`

Создать `src/pages/admin/Dashboard.tsx`:
- **Количество заказов** с фильтрацией по периоду (день/неделя/месяц/3м/6м/12м/произвольный).
- **Количество по статусам** — карточки с числами по каждому статусу.
- **Сумма продаж** с аналогичной фильтрацией по периоду.
- **Быстрые ссылки** — кнопки на Заказы и Товары.
- Данные из Supabase, фильтрация на клиенте по `created_at`.

### 6. Навигация

- Добавить ссылку «Дашборд» в `AdminLayout.tsx` (иконка `LayoutDashboard`).
- Добавить роут `<Route path="dashboard" element={<AdminDashboard />} />` в `App.tsx`.
