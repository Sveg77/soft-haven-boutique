

## План: Генерация фото товаров

У нас 8 товаров без изображений. Нужно сгенерировать фото для каждого и сохранить их так, чтобы они отображались в админке и на сайте.

### Шаги

1. **Создать storage bucket `product-images`** — миграция для создания публичного бакета в Supabase Storage с RLS-политикой на чтение для всех и запись для authenticated.

2. **Сгенерировать 8 изображений** через AI Gateway (Nano banana — `google/gemini-2.5-flash-image`) с промптами на английском для каждого товара:
   - Комплект постельного белья «Лаванда» — lavender bedding set
   - Комплект «Прованс» — Provençal style bedding
   - Подушка «Облако» — soft white pillow
   - Одеяло бамбуковое — bamboo duvet
   - Шторы льняные «Натюрель» — natural linen curtains
   - Покрывало стёганое «Уют» — quilted bedspread
   - Набор полотенец «Вафельный шик» — waffle towel set
   - Декоративная наволочка «Прованс» — decorative pillowcase

3. **Загрузить изображения в storage bucket** и получить публичные URL.

4. **Обновить записи товаров** — записать `image_url` для каждого товара в БД.

5. **Добавить загрузку файлов в админке** — заменить поле «URL изображения» на кнопку загрузки файла, которая загружает фото в storage и сохраняет URL.

### Технические детали

- Генерация через скрипт `lovable_ai.py` с флагом `--image`
- Загрузка в storage через Supabase JS SDK в скрипте или curl
- Публичный URL формируется как `{SUPABASE_URL}/storage/v1/object/public/product-images/{filename}`
- Форма редактирования товара получит `<input type="file">` + превью загруженного изображения
