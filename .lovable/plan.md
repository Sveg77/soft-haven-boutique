

## План: Исправление фото цветов и выравнивание карточек

### Проблема 1: Фото не соответствуют цветам

12 из 18 товаров имеют одинаковое фото для всех 3 цветов (URL дублируется). Уникальные фото есть только у 6 товаров (постельное белье и подушки/одеяла).

Товары без уникальных фото по цветам:
- **Шторы**: блэкаут, льняные, тюль (9 фото нужно = 3 товара x 2 недостающих варианта)
- **Покрывала**: плед, жаккардовое, стёганое (6 фото)
- **Полотенца**: махровое, вафельный шик, кухонные (6 фото)
- **Аксессуары**: саше, наволочка, бохо-подушка (6 фото)

**Итого: 24 новых изображения** для цветовых вариантов.

### Проблема 2: Кнопки не выровнены

Карточки товаров в категории имеют разную высоту из-за длины названий и цен. Кнопка "Купить" и селектор количества не выровнены по нижнему краю.

### Шаги реализации

1. **Сгенерировать 24 изображения** через AI (google/gemini-2.5-flash-image) — для каждого из 12 товаров создать 2 цветовых варианта на основе существующего фото.

2. **Загрузить в storage** bucket `product-images` и получить публичные URL.

3. **Обновить `images` массив** в БД для каждого товара — заменить дубликаты на уникальные URL.

4. **Исправить CSS карточек** в `CategoryPage.tsx` — использовать `flex flex-col h-full` и `mt-auto` для прижатия кнопок к нижнему краю карточки.

### Технические детали

- Карточка `ProductCard`: обёртка получит `h-full flex flex-col`, блок с кнопками — `mt-auto`
- Генерация изображений через скрипт с `lovable_ai.py --image` и промптами вида "Change the color of this [product] to [color], keep the same style and composition"
- Загрузка через curl в Supabase Storage, обновление через SQL
